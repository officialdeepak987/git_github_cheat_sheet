Git & Github Cheat Sheet.

Cheak .git file present or not

By using ls its showing not any file present

=> ls

But when using

=> ls -a

it show .git file by using git init

Git insitial by using command

=> git init

Check status

=> git status

Before commit we have to send file staging area

=> git add (file name)

sending multiple file staging area by using command

=> git add .

After staging area we have to send file for commit

=> git commit -m "meaning full message "

After commit file goes repositery

=> git status

To clear the terminal by using command

=> clear

or

=> command + k

To check how many commit and what we change till now.

=> git log

(get hash code) by using git log

checking status

=> git status

To move one commit back by using

=> git checkout (commit hash code/branch name)

To come from original file

=> git checkout master

Note

checkout command basically use to move from
one branch to another or vice versa.

Git

  • Git is a distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
  • Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
  • Git is used by some of the largest software projects in the world, including the Linux kernel, Android, and Google Chrome.
  • Git is designed for speed and efficiency, and it is able to handle projects of any size. Git is also a distributed version control system, which means that every user has a complete copy of the project history. This makes Git very resilient to data loss, and it also allows for offline work.
